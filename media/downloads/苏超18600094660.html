项目组存储过程触发的ORA-7445告警的紧急处理

软件环境：红旗linux AX4sp3，Grid 11gR2，Oracle 11gR2

硬件环境：HP DL980

责任描述：1、sqlplus登陆数据库查看数据库alert.log trace 日志定位问题。
2、根据报错ora参数与mos文档参考比对。
3、与应用处人员沟通ora报错时的操作。
4、找到解决方法需要大PSU5以上版本补丁。
5、与客户沟通预约停机时间进行数据库打补丁。

项目简介：某银行信用卡业务系统数据库ora-07445告警，客户公司对于数据库异常告警较为敏感，因此紧急协调人员进行问题分析与处理。
处理过程：
首先查看数据库alert.log日志发现：Errors in file /oracle/diag/rdbms/goldrpts/goldrpts1/trace/goldrpts1_ora_1275.trc  (incident=538476):ORA-07445: exception encountered: core dump [kkdcgcr_d()+2359] [SIGBUS] [ADDR:0x100000044] [PC:0x1A979D7] [Non-existent physical address] []，深入到incident trace文件中进行检查，发现为执行alter table disable constraint的时候触发的ORA-7445事件，在该时段，实例1的AWR报告的等待事件，可以看到，”RAC referential constraint parent lock”等待突出。
确定原因：
(1)ORA-7445里报告了kkdcgcr()函数。
(2)非空闲等待事件”RAC referential constraint parent lock”突出。
(3)由于跑批处理时用户GESSRPT调用隐含参数P_SYS_MANAGE_USER_CONSTRAINTS时引发alter命令删除一个正在使用的外键而导致的7445事件，也是oracle的一个bug。
解决方案：
1.与MOS 文献Bug 19626063 - ORA-7445 [kkdcgcr] in RAC with fix 13916228 (Doc ID 19626063.8) 描述的bug一致，协调停机时间打psu5补丁。
2.与应用处协商以后杜绝disable表中的约束操作，尽量避免发生此类告警。
