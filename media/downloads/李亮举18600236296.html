2014/10 - 2014/10

系统出现大量数据积压故障处理

责任描述：系统出现大量积压现象。数据库处在挂起状态。无法正常运行。需要处理使业务恢复正常工作

项目简介：检查数据库状态发现数据库没有DOWN机。发现数据库视图中SESSION＿WAIT中存大量的ENQUEUE， LIBRARY　CACHE　PIN，LIBRARY　CACHE　LOCK等待事件。断定是由于系统中存在着某个阻塞。导致数据库处在等待的状态。通过查询相关视图及分析，最后确定为数据库在对一视图进行ALTER　编译操做时意外终止，对该视图产生锁死。其他进程无法对该视图进行操做，并且应用程序每执行一次业务时都对该视图做预编译。从而产生了大量的等待，后将阻塞进程杀掉。系统恢复正常。后来经核对，是由于业务人员当时对该视图中所关联的表的结构进行了修改。执行alter table name add column操做。之后没有对该视图进行手工重新编译，造成此次问题
