项目简介：呼叫中心主库使用oracle 10g,aix系统，使用nmon_analyser分析呼叫中心服务器性能，发现系统使用高峰期在每天的9点到17点。使用awr日志分析高峰期数据库性能，发现Top 5 Timed Events中最高两项等待事件为db file sequential read、db file scattered read，未发现其他可疑等待事件。查看SQL Statistics，并分析排名靠前的sql语句，未发现执行计划不合理。可以判断是io性能问题，得出结论：1.表设计不合理，有些超过4g的大表未做成分区表；2.大量历史数据未清理；3.主库上的dsg同步占用大量io资源。和业务部门沟通，清理历史数据，并将某些表在线重定义成分区表解决问题，DB Time下降20%，未再接到坐席中心投诉。
