2015/07 - 2015/08

武汉某公司的数据库性能优化

软件环境：Linux5.3 ORACLE11g

硬件环境：IBM P550 服务器、IBM 光纤磁盘阵列

责任描述：

项目简介：
1.收集统计信息，观察 SQL 执行计划，发现走了索引，表之间索引关系正常；
2.查看 AWR 的建议，在查询过程中，用 AWR 每隔 10 分钟生成一个报告，获得三份报告，观察报告发现TOP5 等待事件排名第一的是 DB CPU，同时time model 中 sql execute elapsed time 占了 70%，观察table fetch continued row 发现数据库行迁移情况严重；
3.选择深夜业务空闲时，新建表再重新插入数据，增大该表 PCTFREE 值，核查新表的行迁移情况；
4.通过调优，对比第一次和最后一次产生的 AWR 报告，分析 table fetch continued row 行迁移明显减少,通过 Load Profile 发现采用逻辑读的次数减少，最大的提升在于语句执行的速度，提高65%，执行的时间恢复到十五分钟左右,满足了客户需求；
5.分析 time model 发现，通过 Segments by Physical Reads 物理读减少了19.65%；
